= 了解授權和訂閱的計費方式
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
計費偏好設定可讓您確定如何處理有效許可證和訂閱的付款和續訂行為。您可以全域套用這些偏好設置，使其影響所有計費關係；或者，當特定合約需要單獨配置時，也可以在單一訂閱層級套用這些偏好設定。

* *市場訂閱* 對應於雲端市場合約（AWS、Azure 或 Google Cloud），使用費透過超大規模帳戶收取。
* *直接許可證* 指的是直接從NetApp購買的許可證，透過許可證金鑰、發票接收人和續約選項在NetApp Console中進行管理。


您可以根據採購模式配置組織如何分配和支付服務費用。



== 權限

NetApp Console中的授權和訂閱管理是透過使用者角色來控制的。這些角色提供不同的存取級別，啟用或限制使用者對此功能可執行的特定操作。

[cols="2,5"]
|===
| 角色 | 描述 


| *行政*  a| 
擁有此角色的使用者對許可證和訂閱擁有完全控制權。管理員使用者可以：

* 查看所有授權和訂閱。
* 新增許可證。
* 將訂閱與帳戶關聯。
* 修改現有許可證詳情。
* 刪除許可證或取消訂閱。




| *檢視器*  a| 
擁有此角色的使用者只有唯讀權限。查看器用戶可以：

* 查看所有授權和訂閱。


|===


== 帳單偏好

計費偏好設定可確保正確套用使用費和續費。正確配置可以減少計費糾紛，並更容易管理結合了NetApp自有容量和市場購買容量的混合授權模式。

例如，全域計費偏好設定可以統一您的發票聯絡人列表，以便所有服務共用一個計費管理員。您也可以使用成本中心標籤來統一跨部門或地區的用量，並啟用自動續約以防止服務中斷。新的Cloud Volumes ONTAP (CVO) 部署會自動繼承您在此處定義的計費模式，從而在新增系統時保持您組織的計費模式的一致性。

計費首選項在NetApp Console的「Licenses and subscriptions」部分中進行設定。

從那裡，您可以：

* 為您的組織設定帳單偏好
* 啟用自訂 CVO 配置以進行進階映射
* 編輯或更新市集訂閱詳情




== 欄位

計費欄位儲存有關每個授權或訂閱計費方式的元資料。它們描述了誰需要付費、續費方式以及每項服務所屬的帳戶或成本中心。



=== 市場領域

市場欄位描述了源自雲端提供者（例如 AWS、Azure 或 Google Cloud）的合約。這些欄位定義了NetApp Console如何將您的訂閱與儲存在提供者市場中的計費資訊連接起來。他們也控制續約、區域分配和帳單帳戶的管理方式。

* *市場提供者* — 標識處理計費的雲端平台。服務提供者決定支援的功能、合約類型和續約規則。例如，AWS Marketplace 訂閱採用基於帳戶的計費方式，而 Azure 可能同時支援基於帳戶和基於租用戶的計費模式。
* *帳單帳戶或訂閱 ID* — 指定用於處理費用的帳戶的唯一識別碼。這樣可以確保使用資料計入正確的雲端計費方案。
* *合約期限* — 定義訂閱期限和續約設定。




=== 直接許可字段

直接許可欄位適用於直接購買的NetApp管理合同，而不是透過雲端市場購買的合約。它們定義了NetApp如何向您開立發票、費用發送到哪裡以及發票上顯示的法律實體。這些欄位對於使用採購訂單或集中式公司帳單的組織至關重要。



=== Keystone田

Keystone欄位適用於NetApp Keystone訂閱協議涵蓋的使用情況。它們允許您將按計量服務使用情況與您的Keystone計費設定檔連接起來，使訂閱資料與其他基於容量的授權模式保持一致。

* *協定 ID* — Keystone合約的唯一識別碼。它將NetApp Console中的計費記錄與您的Keystone協定關聯起來，並允許同步使用情況報告。
* *服務範圍* — 列出Keystone協定涵蓋的服務或容量池。此欄位明確合約中包含哪些工作負載或部署，並確保只有符合資格的資源才能透過Keystone計費。




== 計費選項和定義

計費選項定義了當連接多個計費來源時， NetApp Console如何決定使用費用的優先順序並套用這些費用。它們控制哪個帳戶先收費，以及在達到容量限制時如何在NetApp授權和市場訂閱之間切換計費方式。



=== 條款和選項

您可以根據貴公司的採購策略選擇不同的計費方式：

* * NetApp許可證優先* — 優先透過有效的NetApp授權進行計費。當這些許可證的容量耗盡時，使用量會自動轉移到關聯的市場訂閱。
* *僅限 Marketplace 訂閱* — 所有計費均透過 Marketplace 合約進行，繞過NetApp授權。對於透過雲端服務提供者計費來規範採購的組織來說，這種情況很常見。
* *按需付費（PAYGO）* — 僅對實際使用量收費。
* *年度合約*——鎖定固定的訂閱期限。
* *自動續約* — 允許許可證或市場合約到期時自動續約。停用此選項需要手動續訂，以防止服務中斷。
* *自訂 CVO 配置* — 允許將多個市場訂閱對應到單一超大規模資料中心。當不同的業務部門或專案需要在同一雲端環境中擁有不同的計費帳戶時，這可能會很有用。




=== 典型配置路徑

以下範例說明了不同的計費方式如何與常見情況相對應：

|===
| 設想 | 推薦的計費方式 


| 貴組織直接購買了NetApp容量。 | *優先考慮NetApp授權* 


| 所有使用量均透過市場合約計費。 | *僅限商城訂閱用戶* 


| 您可以在同一雲端服務提供者下為多個項目計費。 | *啟用自訂 CVO 配置* 


| 您希望恢復到簡單的單一來源計費設定。 | *停用自訂 CVO 配置* 
|===
例如，考慮一個同時擁有NetApp直接授權和市場訂閱的組織。典型的配置會將「NetApp許可證優先」設定為預設計費模式，以便在市場計費之前使用預付費容量。然後，在*自訂 CVO 配置*下，每個超大規模雲端服務供應商（AWS、Azure、Google Cloud）都會被指派其對應的訂閱 ID。

儲存這些設定後，管理員可以在「Licenses and subscriptions」下查看映射，以驗證每個服務是否指向正確的帳戶。為了進一步驗證，他們可以將NetApp Console中的使用情況資料與雲端計費儀表板進行比較，並根據需要調整成本中心標籤或聯絡人，以保持報告的一致性。
